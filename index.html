My Process

1

Understand

Explore the user and the problem — needs, context, pain points. No design without understanding first.

2

Define

Structure the problem — user flows, information architecture, key goals. Clarity before creativity.

3

Wireframe

Low-fidelity sketches to explore layout and logic without getting distracted by visuals.

4

Design

Clean interfaces in Figma — spacing, typography, visual hierarchy to guide the user naturally.

5

Iterate

Refine through feedback and testing. Good design improves through iteration.
